# Railway Deployment Guide

## Prerequisites
- GitHub account with your repository
- Railway account (https://railway.app)

## Step-by-Step Deployment

### 1. Sign Up on Railway
- Go to https://railway.app
- Sign up with GitHub (recommended for easy integration)

### 2. Create New Project
- Click "New Project"
- Select "Deploy from GitHub repo"
- Connect your GitHub account if prompted
- Search for and select `skin-disease-doctor-consultation` repository

### 3. Configure Environment Variables
Railway will ask you to add environment variables. Click on "Add Variables" and set:

```
DEBUG=False
SECRET_KEY=generate-a-strong-key-here
ALLOWED_HOSTS=your-railway-domain.railway.app
DATABASE_URL=This will be auto-generated by Railway PostgreSQL plugin
CORS_ALLOW_ALL_ORIGINS=False
CORS_ALLOWED_ORIGINS=https://your-railway-domain.railway.app
```

### 4. Add PostgreSQL Database
- In your Railway project dashboard
- Click "Add Service"
- Select "PostgreSQL"
- Railway will automatically set DATABASE_URL environment variable

### 5. Configure Build and Deploy Settings
- Make sure your project is set to Python
- Python Version: 3.10
- Build command: (Leave empty, will auto-detect)
- Start command: `cd django_backend && gunicorn skin_disease_backend.wsgi --log-file -`

### 6. Run Migrations
After the first deployment:
- Go to Railway dashboard
- Click your web service
- Go to "Deployments" tab
- Click on the latest deployment
- In the "Logs" view, wait for deployment to complete
- Then go to "Commands" tab
- Run: `cd django_backend && python manage.py migrate`
- Run: `cd django_backend && python manage.py createsuperuser`
- Run: `cd django_backend && python manage.py collectstatic --noinput`

### 7. Get Your Domain
- Railway will assign a free subdomain like: `your-app-xxxxx.railway.app`
- Update ALLOWED_HOSTS in environment variables with this domain
- Update CORS_ALLOWED_ORIGINS with this domain

### 8. Access Your App
- Your app is now live at: `https://your-app-xxxxx.railway.app`
- Admin panel at: `https://your-app-xxxxx.railway.app/admin`
- API at: `https://your-app-xxxxx.railway.app/api`

## Important Notes

### Static Files
- WhiteNoise is configured to serve static files
- Media files are stored in `/media` directory (temporary)
- For production, consider using AWS S3 or similar

### Database
- PostgreSQL is used in production (SQLite only for development)
- All migrations will run automatically

### Debug Mode
- Keep DEBUG=False in production
- Set SECRET_KEY to a strong random value

### Custom Domain (Optional)
After deployment, if you want to use a custom domain:
1. Go to Railway project settings
2. Add custom domain
3. Update your DNS records as per Railway's instructions
4. Update ALLOWED_HOSTS and CORS_ALLOWED_ORIGINS with new domain

## Troubleshooting

### Build Fails
- Check if all requirements are in requirements.txt
- Ensure Procfile is correctly formatted
- Check logs in Railway dashboard

### App Crashes
- Go to Railway logs and check error messages
- Common issues: Missing environment variables, database connection
- Make sure you ran migrations

### Static Files Not Loading
- Run collectstatic command in Railway terminal
- WhiteNoise should serve them automatically

## Monitoring

- Check Railway dashboard for logs and monitoring
- Set up email notifications for deployment failures
- Monitor resource usage (CPU, Memory, Database)

Good luck! 🚀
